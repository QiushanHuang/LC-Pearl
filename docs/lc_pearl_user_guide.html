LC-Pearl 使用手册

本手册说明如何使用 LC-Pearl v1.0.0 从 LAMMPS dump 中定量分析液晶 mesogen 聚集行为， 并将结果与 OVITO 可视化、拉伸力学量、温度或外力条件关联。LC-Pearl 的目标不是简单数 “距离 cluster”，而是将聚集行为分解为 mesogen contact、domain、pearl 三个层级。

1. 适用对象和基本假设

粒子类型

默认只把 type 1 ellipsoid 作为 mesogen 聚集成员。type 2 sphere 和 type 3 anchor 不进入聚集成员，但可用于端点、链序号、拉伸轴和 topology 支持。

物理问题

用于分析 Gay-Berne 吸引、链连接和外力拉伸竞争时产生的局部束化、珍珠状 bead、connector 和 force-extension 行为。

核心输出

输出时间序列、OVITO label dump、contact segment、threshold prior 图、domain/pearl 诊断和力学相关量。

2. 推荐工作流

最稳定的使用方式是在需要分析的 dump 输出目录中直接运行 portable wrapper。

cd /path/to/dump_output_folder
python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py

首次运行会创建 lc_pearl_preflight/。你需要把原始输入信息放入这个前置文件夹。

文件或目录你需要放什么LC-Pearl 用它做什么
lc_pearl_preflight/lammps/*.in 或 *.lmp原始 LAMMPS input，必须包含 pair_style gayberne 和 type 1-1 pair_coeff提取 Gay-Berne 参数，重算 E-E pair energy，生成 gb_param_source.in
lc_pearl_preflight/lammps/lammps_executable.txtLAMMPS 可执行文件绝对路径用于 run 0 / microstate potential validation
lc_pearl_preflight/topology/*.data可选 LAMMPS data 文件，包含 Atoms/Bonds自动生成 local_pairs.tsv 和 exclude_pairs.tsv，提高链局域关系判断精度
lc_pearl_preflight/validation/verified_potential.json通常无需手写缓存已验证的 Python GB 重算参数；potential 不变时可复用
lc_pearl_preflight/thresholds/global_thresholds.json通常无需手写缓存全局 2D lobe threshold prior；相同体系可迁移复用

运行模式

python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py validate
python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py run
python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py auto

auto 是默认模式：先检查 potential cache，必要时验证；再检查 threshold prior，必要时重建；最后运行主分析。

3. 配置文件工作流

如果你希望固定参数并批量复用，编辑 configs/quick_run.toml 或前置文件夹中的 lc_pearl_config.toml。

cd /Users/joshua/Desktop/MD/LC-Pearl
/Users/joshua/Desktop/MD/venv/bin/python3 lc_pearl_cli.py print-run --config configs/quick_run.toml
/Users/joshua/Desktop/MD/venv/bin/python3 lc_pearl_cli.py run --config configs/quick_run.toml

print-run 只打印命令，不实际运行。建议每次改配置后先看一次 print-run。

5. OVITO 可视化指南

打开 per_file/*_lc_labels.dump 后，常用 Color Coding 字段如下。

字段含义建议用途
lc_cluster仅用于直观显示的中心距 visual cluster id快速看空间团块，不作为最终 robust domain 定义
lc_cluster_sizevisual cluster 中 mesogen 数量看局部密集团块大小
lc_contact_degree该 mesogen 的 accepted contact edge 数找聚集核心和 connector
lc_min_pair_energy该 mesogen 参与的最强吸引 pair energy找强吸引核心
lc_mean_pair_energy该 mesogen 参与 contact 的平均 pair energy看局部吸引环境
lc_max_gb_strength最大 normalized attraction strength识别强 GB contact
lc_mean_gb_strength平均 normalized attraction strength区分孤立强边和整体强吸引区域
lc_domaindomain id看局部 mesogen 束
lc_pearlpearl id看几个 robust domain 是否组成同一个 3D bead
lc_state状态码，表示未归属、weak、robust、pearl-supported 等快速过滤和上色

如果 OVITO 不能同时加载多个 pipeline，先看 label dump 和 cluster envelopes；contact VTK 可以对代表帧单独打开。

6. 如何分析结果

聚集程度随时间

从 aggregation_timeseries.tsv 画 robust_domain_count、weak_domain_count、pearl_count、largest_pearl_fraction、local_edge_fraction、nonlocal_edge_fraction。这些量能反映拉伸过程中“大 cluster 被拉成多个 pearl”的过程。

外力或温度对比

对每个模拟条件分别运行 LC-Pearl，然后比较稳态区间的均值和置信区间。推荐同时报告 L_parallel、Rg_parallel、Rg_perp、S2_force，避免只用 cluster 数解释力学。

pearl-necklace 语言

pearl_count 描述沿链/空间上出现多少 bead-like 团块；domains per pearl 描述一个 bead 内部是否由多个 domain 挤在一起；connector length 描述 bead 之间被拉开的程度。

7. 参数修改建议

目标优先调哪个参数判断依据
阈值太宽，远距离也连起来先检查 global_thresholds.json 和 lobe split 图；必要时提高 gb_on_strengthOVITO contact segments 是否跨过明显空隙
相邻液晶弱聚集被忽略检查 s_excl、lc_state 和 weak domain 输出相邻束应保留为 weak local domain
多个 domain 是否应合成一个 pearlpearl_gap_cut、pearl_min_cross_contacts、pearl_max_aspect_ratio合并后是否仍是 compact bead，而不是细长 connector
运行太慢threshold_prior.global_frame_stride、workers、关闭 debug JSONL先用 stride 10 试算，再对最终代表体系全量算 prior
输出文件太大edge_diagnostics_table = "off"、write_frame_jsonl = false默认已关闭，只有 debug 时开启

8. Debug 顺序

  1. 检查 lc_pearl_preflight/validation/verified_potential.json 是否存在且匹配当前 LAMMPS input。
  2. 检查 lc_pearl_preflight/thresholds/global_thresholds.json 的 schema 和 recommended thresholds。
  3. 打开 gb_strength_vs_p2_stream_lobe_split_dotgrid.png，确认阈值线是否合理切分 2D lobe。
  4. 检查 diagnostics/diagnostic_summary.json，确认主分析实际使用了同一组阈值。
  5. 在 OVITO 中检查代表帧的 lc_domain、lc_pearl 和 contact segments。
  6. 若某帧异常，再临时打开 edge_diagnostics_table = "sample" 或 write_frame_jsonl = true。

9. 什么时候需要重新验证或重建先验

变化是否需要重新 GB validation是否需要重建 threshold prior
LAMMPS pair_style、pair_coeff、单位或 shape 定义改变需要需要
只改变外力大小，但 potential 不变通常不需要同一体系可复用，但建议用代表温度/状态全量 prior
温度范围明显改变不一定建议用覆盖高温到低温的 dump 重建全局 prior
链长、拓扑、special_bonds 改变potential 不变时不一定建议重建，因为 local/nonlocal 结构不同

10. 引用和报告建议

组会或论文中建议用以下结构报告：

  • 体系与输入：粒子类型、GB 参数、拉伸/温度条件、dump 采样。
  • 阈值来源：展示全量 GB strength x P2 lobe split 图和 recommended thresholds。
  • 聚集层级：domain 解决局部束，pearl 解决 3D bead，mechanics 解决力学响应。
  • 结果面板：N_domain(t)、N_pearl(t)、largest pearl fraction、local/nonlocal edge fraction、L_parallel、Rg_parallel/Rg_perp。
  • 可视化：OVITO label + contact segments 的代表帧。
