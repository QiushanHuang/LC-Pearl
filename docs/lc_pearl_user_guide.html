LC-Pearl 使用手册

LC-Pearl 用于定量分析液晶 mesogen 在拉伸、降温或其他动力学过程中形成的局部束化、robust domain 和 pearl-necklace bead。v2.1.0 的正式流程是：先验证 Gay-Berne potential，再由全局 GB strength × P2 二维分布自动生成阈值先验，然后才运行 domain、pearl、力学和 OVITO 可视化分析。

1. 快速开始

进入包含 dump 文件的目录：

cd /path/to/dump_output_folder
python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py

首次运行会创建 lc_pearl_preflight/。把原始 LAMMPS input、LAMMPS 可执行文件路径和可选 topology data 放进去后，运行：

python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py auto

2. 前置文件夹需要什么

路径你放入什么pipeline 用途
lc_pearl_preflight/lammps/*.in 或 *.lmp原始 LAMMPS input，包含 pair_style gayberne 和 type 1-1 pair_coeff提取 GB 参数并生成 gb_param_source.in
lc_pearl_preflight/lammps/lammps_executable.txt一行 LAMMPS 可执行文件绝对路径用于 run 0 / microstate potential validation
lc_pearl_preflight/topology/*.data可选 LAMMPS data 文件，含 Atoms/Bonds生成 local_pairs.tsv 和 exclude_pairs.tsv，提高 local/nonlocal 判断精度
validation/verified_potential.json通常自动生成相同 potential 可复用的验证缓存
thresholds/global_thresholds.json通常自动生成相同体系可复用的阈值先验缓存

3. 配置文件工作流

适合批量分析多个温度、力或链长目录。

cd /Users/joshua/Desktop/MD/LC-Pearl
/Users/joshua/Desktop/MD/venv/bin/python3 lc_pearl_cli.py print-run --config configs/quick_run.toml
/Users/joshua/Desktop/MD/venv/bin/python3 lc_pearl_cli.py run --config configs/quick_run.toml

print-run 只打印命令，不运行；建议每次改配置先执行它。

5. OVITO 上色逻辑

字段含义建议用途
lc_cluster中心距 visual cluster id快速看空间团块，不作为最终 robust domain 定义。
lc_aggregation_tier0 无接触，1 weak/transition，2 aggregation，3 core shoulder，4 strict core推荐第一优先上色字段。
lc_contact_degreemesogen 参与的 accepted contact 数找接触 hub 和 connector。
lc_min_pair_energy最强吸引 pair energy定位强吸引接触。
lc_max_gb_strength最大归一化 GB 吸引强度定位强 GB contact。
lc_core_contact_degreegb_strength ≥ gb_core 且 P2 > 0.71 的接触数看 core-shoulder 区域。
lc_strict_core_contact_degreegb_strength ≥ gb_strict_core 且 P2 > 0.80 的接触数看最强核心区域。
lc_domainweak 或 robust domain id看局部 mesogen 束。
lc_pearl3D pearl id看 bead-like pearl membership。
lc_state0 none，1 weak domain，2 robust domain紧凑显示 domain 状态。

如果 OVITO 不能同时加载多个 pipeline，先打开 label dump；contact segments VTK 可对代表帧单独打开。

6. 如何判断结果可靠

  1. 检查 lc_pearl_preflight/validation/verified_potential.json 是否匹配当前 LAMMPS potential。
  2. 检查 lc_pearl_preflight/thresholds/global_thresholds.json 是否存在，并由代表性数据生成。
  3. 查看 gb_strength_vs_p2_stream_hexbin.png 与 lobe_split_preview/gb_strength_vs_p2_stream_lobe_split_dotgrid.png。
  4. 检查 diagnostics/diagnostic_summary.json，确认主分析应用了同一组阈值。
  5. 在 OVITO 用 lc_aggregation_tier、lc_domain、lc_pearl 核对代表帧。
  6. 看 domain_size_vs_s2.png 和 domain_size_vs_domain_count.png 是否与物理直觉一致。

7. 不同目的如何使用

单条轨迹

看 N_domain(t)、N_pearl(t)、largest pearl fraction、local/nonlocal edge fraction 和 OVITO 代表帧。

外力序列

复用相同 potential cache 和代表性 threshold prior；比较稳态 L_parallel、pearl_count、largest_pearl_fraction。

降温轨迹

用覆盖高温到低温的 dump 建全局 prior；分析 aggregation onset 和 core-contact 增长。

debug

对异常 frame 临时开启 edge_diagnostics_table="sample" 或 write_frame_jsonl=true。
