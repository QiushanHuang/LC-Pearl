LC 参数精度升级方案与完成进度

版本：2026-04-29 final-upgrade。旧的 next-roadmap 和 precision-upgrade 文件保留为过程记录；本文说明当前 pipeline 已完成哪些升级、哪些参数仍是经验/保护参数，以及以后怎样继续提高精度。

一、升级目标

目标不是把所有参数都“自动化”，而是把每个参数分成三类：可由数据自动推荐并安全应用；可由数据推荐但必须人工确认；只能作为保护性固定参数。这样可以避免把统计不足、event-aware 非无偏样本或未验证 potential 误当成精确结论。

1. 输入采样记录 dump、frame、pair、microstate 的候选数、选中数、具体对象和抽样率。
2. Potential 验证whole-frame run 0 加 sampled two-particle microstates，生成 verified potential artifact。
3. 阈值校准2D joint gate 默认启用，bootstrap 和 support gate 决定是否允许 apply。
4. 结构审查domain/pearl 候选表、recommend-only pearl 参数、OVITO 和统计图共同确认。

五、什么时候需要重验 potential

验证一次不是永久有效，而是对同一个 fingerprint 有效。fingerprint 包括 GB 参数文件、forwarded 参数、LAMMPS executable、mesogen type、local/exclude 文件等。

情况是否重验原因
只换同一 potential 下的新 dump / 新 force / 新温度通常不用如果 shape、pair_coeff、pair_style、special_bonds/topology 处理都不变，已验证 artifact 可复用。
换 pair_coeff、pair_style、ellipsoid shape、单位制必须重验GB 能量函数或参数变了。
更换 LAMMPS build 或 executable建议重验不同编译版本/patch 可能影响 pair style 行为。
新增或修改 local/exclude/topology 规则必须重验分析中哪些 pair 参与/排除会改变。
发现 Python/LAMMPS microstate 差值接近 tolerance必须扩大抽样后重验说明 potential reconstruction 还不稳。

六、为什么还需要 topology

topology 模块不是为了把 bond 当成聚集，而是为了正确解释聚集：相邻 mesogen 的吸引要保留，但需要知道它是 local 支持还是 nonlocal 支持；LAMMPS 的 special_bonds 或排除规则也会影响某些 pair 是否真的计算 pair energy。

建议后续把 topology 拆成独立模块：读取 LAMMPS data/in 文件，生成 local_pair_file 和 exclude_pair_file，并把文件 sha256 写入 potential fingerprint。这样同一套 domain/pearl 分析可迁移到其他项目。

七、仍可继续提高精度的方向

方向为什么现在不做何时值得做
严格 2D KDE/GMM posterior boundary当前样本来自筛选和分层抽样，直接做复杂 posterior 容易制造不可解释边界。当 candidate pair 覆盖全量、无偏样本足够大、人工 OVITO 标签可作为 holdout 时。
pearl 参数自动 applypearl 是 domain 的高阶组合，误合并代价大；当前只有 recommend-only 更稳。当多个 force/temperature 下 pearl candidate 表和 OVITO 审查都显示稳定 plateau 时。
全量 microstate validation可能很慢；默认 12 个抽样用于快速 golden check。当需要发表级 potential 复现证明，或发现抽样 pair 有失败趋势时，用 --microstate-sample-percent 100。
跨参数 ARI/Jaccard 稳定性报告需要完整 force/temperature 批处理和统一 holdout 设计。当要把“robust domain/pearl”作为论文主指标时。
使用原则：当前 pipeline 的强项是“可审计、可复现、可在 OVITO 中核对”。不要把推荐阈值当作绝对真值；先看 potential validation、sampling manifest、apply_allowed、warnings，再看 domain/pearl 统计趋势。
