LC-Pearl V2 使用指南

这份指南说明你现在怎么跑 V2、前置文件夹里放什么、阈值先验如何单独运行、如何看输出、如何在 OVITO 里检查 lc_cluster/lc_domain/lc_pearl/lc_state，以及怎样判断结果是否可靠。

1. 推荐工作流

进入 dump 目录当前目录就是要分析的一组 dump。
准备 preflight放 LAMMPS input、executable、可选 topology data。
自动验证没有验证缓存就先跑 run 0 / microstate。
阈值先验没有 global_thresholds.json 就先流式生成。
正式分析生成 V2 output、OVITO labels、diagnostics。
cd /path/to/dump_output_folder
python3 /Users/joshua/Desktop/MD/LC-Pearl/lc_pearl_here.py

第一次运行会创建 lc_pearl_preflight/ 模板并停止。你补齐文件后再次运行。默认输出目录是 lc_domain_pearl_v2_output/。

2. preflight 文件夹要放什么

你要提供的原始文件放置路径LC-Pearl 自动提取/生成
原始 LAMMPS input，含 pair_style gayberne 和 type 1-1 pair_coefflc_pearl_preflight/lammps/*.in 或 *.lmp自动标准化为 lammps/gb_param_source.in，提取 GB potential 参数、special_bonds、units、atom_style。
LAMMPS executable 路径lc_pearl_preflight/lammps/lammps_executable.txt用于 run 0 验证；内容示例：/Users/joshua/Desktop/Qiushan_Code/lammmps_git/lammps/build-asphere-mpi/lmp。
代表帧 dump当前 dump 目录，不放进 preflight默认自动选择第一个匹配 *.dump 的文件；也可在 [validation].representative_dump 指定。
原始 LAMMPS data 文件，可选但推荐lc_pearl_preflight/topology/*.data 或 *.dat自动生成 topology/local_pairs.tsv 和 topology/exclude_pairs.tsv。你不需要手写这两个 TSV。
已验证阈值先验，可选lc_pearl_preflight/thresholds/global_thresholds.json如果 fingerprint 匹配、GB potential 已验证、confidence gate 通过，V2 才会应用它；适合从高温到降温全采样得到一套项目级阈值后复制到多个相似模拟。注意“相似模拟”仍需你从温度、密度、链长、构型分布上做科研判断，代码只能检查 potential 和若干输入 fingerprint。
旧命令映射：旧的 --gb-param-file "../in.single_chain_constant_force_aligned_box-F0.38-tail_v0_no_thermal-xuyu-zu.lmp" 现在等价于把这个 .lmp 文件放进 lc_pearl_preflight/lammps/。旧的 --dump-file traj.force_clamp_aligned.6055000.dump 现在等价于在当前 dump 目录保留这个 dump，并在 config 中可选指定。

4. 阈值先验如何单独运行

如果你想用一个覆盖高温到降温的 dump 文件夹做全采样，生成一套项目级阈值，再复制到其它力/温度模拟，直接运行：

cd /path/to/reference_dump_folder
python3 /Users/joshua/Desktop/MD/LC-Pearl/scripts/lc_threshold_prior.py . \
  --gb-param-file lc_pearl_preflight/lammps/gb_param_source.in \
  --output-dir lc_pearl_preflight/thresholds \
  --global-threshold-file lc_pearl_preflight/thresholds/global_thresholds.json \
  --anchor-types 3 \
  --global-frame-stride 1 \
  --global-frame-budget 0 \
  --workers auto

--global-frame-stride 1 表示全量候选帧；改成 10 就是在整个输入文件夹排序后的全局 frame 序列里每 10 帧取 1 帧，改成 100 就是每 100 帧取 1 帧。对于每个 dump 文件只有一帧的目录，不要用 --every 10 来控制全局抽样，因为 --every 只作用于单个 dump 文件内部。

输出怎么看
global_thresholds.json最终可复用先验。看 recommended.gb_off_strength、recommended.gb_on_strength、recommended.p2_cut、calibration_status、apply_allowed。
streaming_manifest.json说明读了多少文件、多少 raw frames、多少候选 frames、实际 selected frames、worker 数。
selected_frame_manifest.tsv列出 streaming prior 实际使用的每一个 frame，包括文件名、timestep、全局 frame index、stride、offset、frame sampling weight。
stream_hist_2d_gb_p2.tsv全局 2D 统计表，不是所有 pair 明细，因此不会生成几十 GB 数据库。
gb_strength_vs_p2_stream_hist.png最重要的图。图上有 gb_off、gb_on、p2_cut 数值线。
audit_pair_examples.tsv少量代表 pair 明细，用来人工抽查，而不是完整数据库。
可迁移使用：如果同一 potential、同一 mesogen type、同一 topology 排除规则、相近 dump schema 不变，可以把整个 lc_pearl_preflight/thresholds/ 复制到其它 dump 目录复用。V2 会检查 GB 参数 hash、mesogen type、axis、local/exclude pair hash、s_excl 和 r_energy_cap；不匹配会自动重建或报错。
阈值 prior 的 pair 范围：当前 V2 prior 默认统计全部未排除 attractive candidate pairs，同时保留 local/nonlocal 直方图用于审查；它不是 nonlocal-only prior。如果 local contacts 明显主导 2D 分布，需要在 OVITO 和 diagnostics 中确认阈值没有被相邻链段统计带偏。

5. 主输出目录怎么看

文件/目录用途先看什么
lc_domain_pearl_v2_output/run_summary.json复现记录。看 potential_validation、threshold_prior、输入文件数、输出路径。
aggregation_timeseries.tsv主定量结果。N_domain、N_pearl、largest_domain_fraction、largest_pearl_fraction、L_parallel、Rg_parallel/perp。
per_file/*_lc_labels.dumpOVITO 主可视化。用 lc_state、lc_domain、lc_pearl、lc_cluster 上色。
diagnostics/domain_diagnostics.tsvdomain 级审查。看每个 domain 的 size、S2、robust/weak、nonlocal 支持。
diagnostics/pearl_candidate_diagnostics.tsvpearl 合并审查。看两个 domains 是否因为 gap、cross contacts、boundary support、aspect ratio 被接受或拒绝。
diagnostics/*.png快速读图。所有阈值线都会标注具体数值；图只用于判断参数是否明显过宽/过严，不替代 OVITO 抽查。

6. OVITO 上色逻辑

打开 lc_domain_pearl_v2_output/per_file/*_lc_labels.dump。字段名前缀是小写 lc_，在某些字体里看起来像 Ic_。

字段怎么用算法含义
lc_cluster先用它看“肉眼空间聚集在哪里”。视觉辅助 cluster，方便直观看团块，不是最终 robust domain 定义。
lc_cluster_size看每个视觉 cluster 有多大。同一个 lc_cluster 内 mesogen 数。
lc_contact_degree看单个 mesogen 有多少通过阈值的接触边。连接度高通常位于聚集体内部；边界粒子连接度低。
lc_min_pair_energy看每个 mesogen 最强吸引 pair。越负表示吸引越强。
lc_mean_pair_energy看平均吸引环境。比最小能量更平滑，但会受弱边影响。
lc_max_gb_strength看归一化最强吸引。高值说明至少有一个强吸引支持。
lc_mean_gb_strength看平均归一化吸引。用于判断一个粒子整体吸引环境是否稳定。
lc_domain主看局部束。同一编号表示同一帧同一个 domain。编号不保证跨帧连续。
lc_pearl主看空间珠子。同一编号表示同一帧同一个 3D pearl；一个 pearl 可含多个 domains。
lc_state快速区分 weak/robust。0 非 domain，1 weak local domain，2 robust domain。
没有 OVITO Pro 的加载建议：先只打开 *_lc_labels.dump 看粒子标签。*_lc_contact_segments.vtk 是接触线段，OVITO Pro 可以 Add to scene 叠加；没有 Pro 时可以先忽略它，或者单独打开 VTK 看边是否过宽。

7. 如何判断结果可靠

检查项可靠的表现问题信号
Potential validationverified_potential.json status 为 validated，microstate 误差小。status 不是 validated；换了 potential 但还复用旧 cache。
Threshold priorglobal_thresholds.json 中 apply_allowed=true 且 run summary 里 threshold_prior.applied=true，selected frames 足够，2D 图中背景/gray/strong 有可分结构。calibration_status=low；applied=false；2D 图满屏无分界；selected frames 太少；out_of_hist_range_pair_fraction 偏高。
OVITO 抽查lc_pearl 对应肉眼的 3D 珠子，lc_domain 对应局部取向束。远距离结构被同色连成一个 pearl，或明显紧密结构被大量拆碎。
Domain diagnosticsrobust domains 多数 size ≥ 3、S2 达标、有 nonlocal support。robust 全是二聚体或全靠 local edge。
Pearl candidate diagnostics被合并的 domains 同时有小 gap、多接触、两侧边界支持、aspect ratio 不高。只有一条桥接边也合并，或 aspect ratio 很大的细长丝被算成 pearl。

8. 什么时候需要重新验证或重新生成 prior

变化run 0 验证threshold prior
GB 参数或 LAMMPS pair_style/pair_coeff 变了必须重验必须重建
LAMMPS executable 或版本变了建议重验如果验证通过且 potential 相同，prior 可复用。
topology special_bonds/exclude 规则变了建议重验必须重建；local/exclude pair hash 不匹配时 V2 会拒绝复用。
