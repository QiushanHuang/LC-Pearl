LC Domain-Pearl V2 算法与参数说明

V2.1.0 的核心改变是把“阈值推荐先验”拆成独立、可迁移、可缓存的 streaming 模块：先用全局统计或项目级代表统计确定 gb_off/gb_on/p2_cut，并额外给出 diagnostic gb_core/p2_core core-shoulder 层和 gb_strict_core/p2_strict_core strict-core 层，再用这些参数正式生成 domain、pearl 和 OVITO label。这样避免 V1 中“先用默认值分析，分析后才推荐阈值”的逻辑错位。

名称：LC/Ic Domain Pearl V2日期：2026-04-30主定义：3D bead pearl阈值：pre-analysis prior

1. 总流程

Preflight从当前 dump 目录识别原始 LAMMPS input、LAMMPS executable、可选 topology data，并生成标准输入。
GB 验证用代表帧做 LAMMPS run 0 / microstate pair check，生成 verified_potential.json。
Threshold Prior独立 streaming 扫描候选 pair 的 GB strength × P2 分布，写 global_thresholds.json。
Main Analysis先把 prior 推荐值注入正式参数，再计算 contact edge、domain、pearl、mechanics。
Diagnostics输出 edge/domain/pearl 候选表、图、参数来源和 run summary。
OVITO输出带 lc_* label 的 dump，用颜色审查聚集体和边界误差。
V2 默认顺序：threshold_prior.mode="auto" 时，如果 lc_pearl_preflight/thresholds/global_thresholds.json 存在且 fingerprint 匹配，并且 GB potential 已验证，才会应用；如果不存在，先运行 lc_threshold_prior.py 生成；然后才跑正式 domain/pearl 分析。v2.1.0 的 gb_core/p2_core 与 gb_strict_core/p2_strict_core 只作为核心接触审查层，不改变 domain/pearl 主定义。
先验不是物理相边界：V2 threshold prior 是对 attractive candidate pairs 的 GB strength × P2 streaming 直方图做 two-lobe valley/shoulder 选择，属于数据驱动分析参数，不是新的 potential，也不是相变判据。

2. 分析对象

对象进入聚集成员吗用途
type 1 ellipsoid / mesogen是domain、pearl、cluster、GB energy、orientation 的唯一成员。
type 2 sphere否不作为聚集成员；可辅助解析链结构，具体取决于 dump/topology。
type 3 anchor否不作为聚集成员；用于端到端方向和 L_parallel。
topology Bonds / special_bonds否用于生成 local/exclude pair，避免把 LAMMPS 中被特殊处理的 pair 当成普通非局域聚集证据。

3. Pair 级计算

符号公式/来源含义精度边界
r_idump 中 xu,yu,zu，缺失时用 x,y,zmesogen 中心位置。如果 dump 没有 unwrap 坐标，跨周期链会影响空间判断。
u_i四元数旋转 body-frame 长轴mesogen 长轴方向；比较时液晶无头向。依赖 dump quaternion 约定；run0/microstate 验证可间接发现明显问题。
q_score|u_i · u_j|0 到 1 的取向一致性，平行和反平行都为 1。只表示 pair 取向，不表示吸引。
p2_score(3*q_score^2 - 1)/2nematic 二阶序参量 pair 版本；1 平行，约 0 随机，-0.5 垂直。适合液晶无头向，比直接角度更稳定。
U_GBPython 根据 LAMMPS Gay-Berne 参数重算两个 ellipsoid 的 pair potential，负值是吸引。必须通过 LAMMPS run 0 / microstate check 才能称为可信重算。
gb_strengthmax(0, -U_GB / U_well)归一化吸引强度，0 表示无吸引，越高表示越接近该几何下吸引井。用于跨取向/距离比较；不是新 potential，只是标准化后的统计量。
delta_s|s_i - s_j| 或 topology/atom id 推断mesogen 沿链序号间隔。用于区分 local support 与 nonlocal support，不用于删除相邻 mesogen。

4. 阈值 prior 如何产生

lc_threshold_prior.py 是 V2 独立模块。它不再把所有 pair 写成巨大的 gb_candidate_pairs.tsv，而是流式遍历候选 frames 和候选 pair，把统计量累积到 1D/2D 直方图，同时只保留少量 audit_pair_examples.tsv 供人工抽查。

参数默认含义
global_frame_stride1显式全局抽帧步长。1 表示全量候选帧；10 表示按排序后的全局 frame 序列每 10 帧取 1 帧；100 表示每 100 帧取 1 帧。对于“一文件一帧”的 dump 目录，这才是控制每多少个 dump 抽一帧的主要参数。若大于 1，它优先于 global_frame_budget。
global_frame_budget00 表示所有帧都进入 streaming 统计；正数表示在全局 frame 序列上按 seeded stride 抽代表帧。V2 会先统计所有 dump 的真实 frame 数，而不是每个文件各抽 48 帧。具体选中帧写入 selected_frame_manifest.tsv。
every1单个 dump 文件内部的帧间隔。若每个 dump 只有一帧，every=10 不会跳过文件；应使用 global_frame_stride=10。
gb_bins/p2_bins120/1202D 直方图分辨率。
gb_strength_max1.5不是单纯绘图范围，而是 streaming histogram 的 GB strength 上限。超过范围的权重会记录在 out_of_hist_range_pair_fraction；当前实现会写出 warning 供人工审查，但不会单独用这个比例阻止 apply。若比例过高，应增大上限后重建 prior。
audit_example_pairs5000只保留这些代表 pair 作为审查样本，不影响直方图全局统计。
workersauto多进程按文件并行处理；大目录会显著加速。

  --anchor-types 3 \

  --global-frame-budget 0 \
  --workers auto
为什么 2D joint 更重要：聚集 contact 不是“能量强”或“取向好”单独成立，而是两者共同成立。V2 在 GB strength × P2 平面上选择 strong/gray gate，避免把强吸引但取向乱、或取向一致但没有吸引的 pair 误算成聚集。
local/nonlocal 语义：当前 V2 prior 默认使用全部未排除 attractive candidate pairs 来估计 pair 阈值，同时输出 local/nonlocal 直方图供审查；它不是 nonlocal-only prior。local edge 不会被删除，但在 domain 阶段只能作为 local support/attach/weak 证据，不能单独代表非局域 pearl 聚集。
并行上限：V2 的自动并行统一按 min(可用 CPU 核数, 输入文件数或任务数, 默认上限 10) 解析。主分析/GB audit 受 LC_PEARL_MAX_WORKERS 控制，threshold prior 受 LC_PEARL_MAX_AUTO_WORKERS 控制，bootstrap calibration 受 LC_PEARL_MAX_AUTO_CALIBRATION_WORKERS 控制。

5. 阈值选择算法

阈值算法来源解释
p2_cut对 streaming P2 边际直方图找 two-lobe valley；若统计不清楚，回退到 weighted median；最后 clamp 到 0.50-0.75。区分取向随机/弱一致/强一致，避免统计噪声给出过低或过高的取向 gate。
gb_off在通过 p2_cut 的 high-P2 pair 上取弱接触 lobe 的保守 shoulder，作为 gray/support contact 下限。太低会把远处弱吸引或噪声连起来；太高会漏掉边界 mesogen。
gb_on在 high-P2 GB 直方图上用 two-lobe valley 分开 weak-contact lobe 与 strong-attraction lobe；若 lobe 不清楚，使用 weighted quantile fallback，并强制 gb_on > gb_off。strong edge 下限。用于 seed domain 的强证据。
p2_core = 0.71固定 core-shoulder pair alignment gate，等价于 |u_i·u_j| > 0.898，约小于 26° 的平行/反平行偏差。只用于 core-shoulder contact 可视化和统计，不改变 robust-domain 或 pearl 判定。
gb_core在 P2 > 0.71 且 GB ≥ gb_on 的强取向强吸引 lobe 中取左侧 shoulder；若没有足够 high-P2 strong-lobe 权重，则回退到 max(current_gb_core, gb_on) 并标记低可信。标记强接触团的 core shoulder。它是“核心证据层”，不是新的 domain seed 阈值。
p2_strict_core = 0.80固定 strict-core pair alignment gate，等价于更接近平行/反平行的取向要求。只用于最强核心接触可视化和统计。
gb_strict_core在 P2 > 0.80 且 GB ≥ gb_on 的高取向强接触团中取 weighted q25，并强制 gb_strict_core ≥ gb_core。标记更严格的 strong core 子集，不改变 domain/pearl 主判定。
robust_min_s2当前仍主要由配置/diagnostics 审查，不由 prior 自动覆盖。domain 内部取向一致性阈值。它依赖 domain 结果而不是 pair prior，V2 暂不把它和 pair 阈值强行耦合。
pearl_gap_cut 等 pearl 参数recommend-only，由 pearl candidate diagnostics 估计。pearl 需要 3D gap、跨域接触、边界支持、aspect ratio 同时审查，目前不自动 apply。
经验参数还存在：n_min=3、s_excl=1、pearl_min_cross_contacts=2、pearl_min_boundary_particles=2、pearl_max_aspect_ratio=3.0 仍是物理定义参数，不是 GB potential 能自动推出的常数。V2 做的是把 pair 阈值从经验默认值升级为数据驱动先验，并把其它参数全部显式写进 diagnostics 和文档。

6. Domain 与 weak/robust 规则

相邻 mesogen 的聚集必须保留。V2 不把相邻接触删除，而是把它们标为 local support；local support 可以参与 weak local domain，也可以附着到已有 seed。是否升级 robust 要看更强证据，而不是只靠二聚体相邻接触。

类别规则物理含义
weak local domain局部/相邻接触形成的短束，或证据不足的局部结构。保留“相邻液晶确实有吸引”的事实，但不让链连接天然靠近主导 robust 统计。
robust domain尺寸至少 n_min，内部 S2 达标，证据数达标，并且需要 nonlocal support 或 persistence 之一；参数稳定性主要作为辅助证据。可以作为 pearl 组成单元和力学关联对象。
ambiguous shell参数扰动下归属不稳定的 mesogen。提醒当前边界态多，定量结论需要结合 OVITO 和 sensitivity 图审查。

7. Pearl 为什么不是大 domain

domain 是接触图中的局部束；pearl 是一个或多个 robust domains 在 3D 空间上形成的 bead-like 致密区。一个 pearl 可以包含多个 domains，因为你肉眼看到的问题正是“几个中等 cluster 挤在一起，到底算一个空间珠子还是多个束”。

pearl 合并条件字段为什么需要
3D gap 足够小pearl_gap_cutpearl 的身份先由空间珠子决定，而不是只按轴向投影。
跨 domain 接触对足够多pearl_min_cross_contacts避免一条桥接边把两个分离束误合并。
两侧边界支持足够pearl_min_boundary_particles要求两个 domains 都有多个边界 mesogen 支持合并。
合并后不细长pearl_max_aspect_ratioaspect ratio 太大说明更像 connector/被拉伸的丝，而不是 bead。
轴向密度辅助诊断用于描述 pearl-necklace 排列、数量、间距、connector 长度，不作为唯一主定义。
