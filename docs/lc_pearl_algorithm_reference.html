LC-Pearl v2.1.0 算法与参数参考

本文定义 LC-Pearl 当前发行版的正式聚集算法。核心设计是把液晶聚集拆成 pair contact、domain、pearl 和 mechanics 四个层级：contact 负责能量和取向，domain 负责局部 mesogen 束，pearl 负责 3D bead-like 珍珠，mechanics 负责拉伸响应。

1. 输入对象

  • type 1 ellipsoid 是默认 mesogen 成员。
  • type 2 sphere 和 type 3 anchor 不进入聚集成员，但可用于链序、端点、拉伸轴和 topology 支持。
  • 每个 mesogen 使用 unwrapped position r_i=(xu,yu,zu) 和 quaternion 计算长轴方向 u_i。
  • 长轴无头尾，因此 pair orientation 使用 |u_i·u_j|。

3. Gay-Berne energy 与 GB strength

生产流程推荐 contact_mode="gayberne"。LC-Pearl 从 LAMMPS input 中提取 Gay-Berne 参数，并对 type 1-1 ellipsoid pair 重算：

U_GB,ij
U_well,ij
gb_strength_ij = max(0, -U_GB,ij / U_well,ij)

gb_strength 是归一化吸引强度。若 U_GB 为排斥或接近零，score 为 0；若 pair 接近该取向下势阱深度，score 接近 1。

Python GB 重算必须由 LAMMPS run 0 / microstate validation 支持。若 potential 参数改变，必须重新验证并重建 threshold prior。

4. 取向量 P2

q_ij = |u_i dot u_j|
P2_ij = (3 q_ij^2 - 1) / 2

P2=1 表示平行或反平行，P2=-0.5 表示垂直。P2 比单纯 |dot| 更适合和液晶二阶取向序参量对应。

5. 2D lobe threshold prior

阈值从完整或 strided candidate pair 的二维分布 gb_strength × P2 得到，不使用已经筛选后的 accepted edges。

参数来源用途
p2_cutP2 分布中低取向背景和高取向 lobe 的分割主 contact orientation gate
gb_off_strength弱接触 lobe 的保守 shouldergray/support edge
gb_on_strength高 P2 条件下弱接触 lobe 与强吸引 lobe 的 valleystrong edge
p2_core_cut=0.71固定 high-orientation 诊断 gatecore shoulder 可视化
gb_core_strength强接触 lobe 的 left shouldercore shoulder GB gate
p2_strict_core_cut=0.80更严格 high-orientation 诊断 gatestrict core 可视化

gb_core 和 gb_strict_core 是诊断层，不改变 robust domain 或 pearl 的主定义。

6. Contact graph

strong edge: gb_strength >= gb_on_strength  and P2 >= p2_cut
gray edge:   gb_strength >= gb_off_strength and P2 >= p2_cut

delta_s 表示链序差。delta_s <= s_excl 的 pair 是 local support。local support 不被删除；它被保留为 weak/local evidence，但不能单独无条件代表非局域 robust aggregation。

7. Domain 分类

domain 是 contact graph 中的局部 mesogen 束。LC-Pearl 输出 weak domain 和 robust domain。

证据典型参数含义
sizen_min=3不是单纯二聚体
orientationrobust_min_s2=0.70内部取向一致
persistencedomain_min_lifetime=2超过瞬态噪声尺度
nonlocal supports_excl 与 topology至少有非相邻 mesogen 支持
parameter stability扰动扫描阈值扰动后仍稳定存在

weak local domain 是有意义的输出，尤其适合解释相邻 mesogen 的局部束化和过渡态。

8. Pearl 合并

pearl 不是“大 domain”。domain 解决局部束，pearl 解决多个 robust domains 是否在 3D 空间中构成同一个 compact bead。

  • min_gap <= pearl_gap_cut：空间间隙足够小。
  • cross_contacts >= pearl_min_cross_contacts：跨域接触足够多。
  • boundary_support >= pearl_min_boundary_particles：两边边界都有支持。
  • aspect_ratio <= pearl_max_aspect_ratio：合并后仍像 bead，不像细长 connector。

11. 参数与精度边界

论文级使用时，应报告版本、potential validation 状态、threshold prior 数据来源、gb_off/gb_on/p2_cut、core/strict-core gates、robust evidence 参数、pearl merge 参数，以及是否使用 topology data。

若缺少 topology data，pipeline 仍可运行，但 local/nonlocal 支持会更多依赖链序和 id 推断；此时对 nonlocal support 的结论应谨慎表述。
