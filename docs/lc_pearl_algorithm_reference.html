LC-Pearl v1.0.0 算法与参数参考

本文给出 LC-Pearl 当前发行版的正式算法定义。核心思想是把“液晶聚集”拆成三个互不混淆的层级： pair contact、mesogen domain、3D pearl。这样既保留相邻液晶的局部束化，又避免把链连接诱导的近邻 误判为真正的 robust aggregation。

1. 分析对象

每帧从 LAMMPS dump 中读取粒子 id、type、unwrapped position、quaternion、shape axes、box 和可选力学列。默认设置为：

  • mesogen_type = 1：type 1 ellipsoid 是聚集成员。
  • anchor_types = 3：anchor 用于端点和拉伸方向，不进入聚集成员。
  • type 2 sphere 可参与拓扑或链序重建，但不作为 domain/pearl 成员。

每个 mesogen 的长轴方向 u_i 由 quaternion 旋转 body-frame axis 得到，pair orientation 使用 head-tail 对称量 |u_i dot u_j|。

关键设计是：阈值必须在主分析前由全量 candidate pair distribution 得到，而不是先用经验阈值跑完后再做 post-run audit。

3. Gay-Berne pair energy 与 normalized attraction

在 contact_mode = "gayberne" 下，LC-Pearl 用 LAMMPS input 中的 Gay-Berne 参数重算 type 1-1 ellipsoid pair energy。每个候选 pair 得到：

U_GB,ij
U_well,ij
gb_strength_ij = max(0, -U_GB,ij / U_well,ij)

gb_strength 是无量纲吸引强度。它把不同取向导致的 well depth 归一化，使“强吸引”不只取决于绝对能量，也考虑该取向下的可达势阱深度。

GB 重算依赖 dump 中的 position、quaternion、shape axes，以及 input 中的 pair parameters。若 potential 设置改变，必须重新验证并重建 threshold prior。

4. Pair orientation score

LC-Pearl 使用二阶 Legendre order score：

q_ij = |u_i dot u_j|
P2_ij = (3 q_ij^2 - 1) / 2

使用绝对点积是因为 Gay-Berne ellipsoid 长轴无头尾方向。P2 = 1 表示平行或反平行，P2 = -0.5 表示垂直。

5. 2D lobe threshold prior

阈值先验从全量 candidate pairs 的 gb_strength x P2 二维直方图中生成。它不依赖已筛选的 edge table，也不依赖 post-run candidate-pair TSV。

5.1 输入分布

对每个候选 E-E pair，若 gb_strength > 0 且未被 topology exclusion 排除，就将其累积进 2D histogram。默认全量帧采样：global_frame_stride = 1。

5.2 阈值定义

参数算法来源物理含义
p2_cutP2 marginal 中低取向背景与高取向 lobe 的 valley，默认最小域 gate 为 0.5pair 需要达到的取向一致性
gb_on_strength在 P2 >= p2_cut 切片中，弱接触 lobe 与强吸引 lobe 之间的 GB valleystrong edge 的吸引阈值，是 robust domain 的主要 pair-level 能量门
gb_off_strengthgb_on 左侧 lobe 的保守 shouldergray/support edge 阈值，用于保留弱支持而不是直接造 robust core

该算法的一个重要约束是：同一张 2D histogram 不应因为旧的 current threshold 不同而产生不同推荐值。current threshold 只记录 provenance，不参与 lobe split。

如果图像没有明显双 lobe，算法仍会给出数值，但应在组会或分析报告中将其标为低结构可分性，而不是强行解释为清晰 pearl transition。

6. Contact graph

每帧构造 mesogen pair graph。边分为 strong edge 和 gray/support edge：

strong edge: gb_strength >= gb_on_strength and P2 >= p2_cut
gray edge:   gb_strength >= gb_off_strength and P2 >= p2_cut

center-distance 或 ellipsoid-distance 模式保留在代码中用于 exploratory analysis，但 v1.0.0 的推荐学术流程使用 Gay-Berne energy。

7. Local support, weak domain, robust domain

delta_s 表示两个 mesogen 的链序号差。delta_s <= s_excl 的 pair 被视为 local support。LC-Pearl 不删除相邻液晶聚集，而是避免只靠相邻键连关系无条件制造 robust domain。

对象定义目的
weak local domain局部相邻或弱支持形成的 mesogen 束保留相邻液晶的真实局部束化信息
robust domain满足 size、orientation、persistence、nonlocal support、parameter stability 等证据的 domain用于后续 pearl 合并和稳定聚集统计

典型 robust evidence 包括：

  • size >= n_min，默认 3。
  • domain S2 >= robust_min_s2，默认 0.70。
  • age >= domain_min_lifetime，默认 2 个处理帧。
  • 至少存在非局域 support contact。
  • 在更严格参数下仍保留足够重叠的 stable core。

8. Pearl 定义

pearl 不是一个更大的 domain。domain 回答“局部 mesogen 束是否存在”；pearl 回答“一个或多个 robust domain 是否在 3D 空间中组成同一个 bead-like compact region”。

两个 robust domains 合并为同一 pearl 需要满足：

  • min 3D gap <= pearl_gap_cut。
  • cross contact count >= pearl_min_cross_contacts。
  • 两侧 boundary particles 都有支持，避免单根桥接边造成假合并。
  • 合并后的 3D aspect ratio 不超过 pearl_max_aspect_ratio，默认 3.0。

aspect ratio 是空间紧凑性的几何约束。若合并后长轴远大于短轴，结构更像 connector 或 stretched bundle，而不是 bead。

9. Mechanics layer

LC-Pearl 同时计算力学相关量，用于将聚集与外力或温度条件联系起来：

量含义用途
L_parallel沿拉伸方向的端到端投影长度主力学响应量
Rg_parallel沿拉伸方向的回转半径分量辅助形状量
Rg_perp垂直拉伸方向的回转半径分量反映横向收缩/团聚
S2_forcemesogen 长轴相对拉伸方向的取向序判断拉伸诱导取向

若有多个恒力模拟，可比较 Delta <L_parallel> / Delta F 作为等效顺应性指标。Rg 不能替代 L_parallel。

11. 精度边界

LC-Pearl 的精度由三部分决定：

  • GB potential 重算是否通过 LAMMPS run 0 / microstate validation。
  • threshold prior 的 2D lobe 是否在全局 candidate-pair 分布中清晰可分。
  • topology/local/exclude pair 是否正确反映本体系的 chain connectivity 和 special-bonds 语义。

如果缺少 topology data，LC-Pearl 仍可运行，但 local/nonlocal 支持会更多依赖链序推断和 id gap。对于论文级结果，建议提供可靠 LAMMPS data 文件。

12. 结果解释原则

  • 不要把 lc_cluster 当作最终物理 domain；它是 OVITO 直观聚集辅助。
  • 不要把 pearl 当作大 domain；pearl 是 3D bead-level object。
  • 不要只报告 N_pearl；同时报告 largest pearl fraction、domains per pearl、local/nonlocal edge fraction 和 mechanics。
  • 不要用 post-run accepted-edge diagnostics 重新选阈值；阈值来自 pre-main-analysis full 2D streaming prior。
