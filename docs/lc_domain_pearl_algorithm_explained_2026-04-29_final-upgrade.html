LC Domain / Pearl Pipeline 最终升级版算法说明

版本：2026-04-29 final-upgrade。旧版 HTML 保留为历史讨论记录；本文是当前代码状态的说明，重点解释 domain、pearl、GB 阈值、抽样审计和 LAMMPS run 0 验证之间的关系。

type 1 ellipsoid 为聚集成员 GB energy contact 2D joint calibration 默认启用 event-aware 可用但禁止自动 apply pearl 参数 recommend-only

一、总体目标

这套代码不是简单数“距离 cluster”。它要回答三件事：液晶 mesogen 之间是否有真实吸引接触；这些接触是否形成局部取向一致的 domain；多个 domain 在 3D 空间中是否组成一个 bead-like pearl，并进一步统计拉伸下 pearl-necklace 的演化。

1. Dump 输入读取 LAMMPS dump，只把 type 1 ellipsoid 当作 mesogen 成员；type 2/3 用于链序和端点辅助。
2. Pair Audit计算每对 mesogen 的 GB energy、gb_strength、p2_score、delta_s，输出候选 pair 表。
3. Threshold Calibration用 2D joint gate 在 GB strength × P2 空间推荐 gb_on/off 与 p2_cut。
4. Domain / Pearl先构造 local/robust domain，再把 robust domains 按 3D bead 规则合并为 pearl。
5. Diagnostics输出 labeled dump、sampling manifests、diagnostic TSV/JSON、plot，用 OVITO 和统计图共同审查。

二、核心对象

对象定义为什么需要
mesogentype 1 ellipsoid，一个液晶刚性椭球中心，带四元数取向。所有聚集判断的基本粒子。
domain由 mesogen pair contact 连接出的局部取向束。可以是 weak local，也可以是 robust。回答“哪些液晶在局部形成束化结构”。
weak local domain主要由相邻或局部链段接触支持，保留但默认不当作强聚集证据。避免丢掉相邻液晶真实吸引，同时不让链连接天然把全链串成一个大 cluster。
robust domain满足尺寸、取向、非局域支持、持续性或参数稳健性等条件的 domain。作为 pearl 合并和力学关联的更可靠结构单元。
pearl一个或多个 robust domain 在 3D 空间中形成的 bead-like 致密团块。回答“几个中等 cluster 挤在一起时，到底是不是同一个空间珠子”。

三、pair contact 的数值

每对 mesogen 计算三个主量：

量公式 / 来源解释
U_GB由 Python 按 LAMMPS Gay-Berne 参数和源码审查后的公式重算。负值表示吸引。最终精确性依赖 lc_lammps_run0_validate.py 的 run 0 校验。
gb_strengthmax(0, -U_GB / U_well)把不同取向/距离下的吸引能归一化成 0 到约 1 的强度；越大代表越接近该 pair 几何下的吸引井。
p2_score(3|u_i·u_j|^2 - 1) / 2无头向液晶常用二阶取向序参量。平行/反平行接近 1，随机附近约 0，垂直接近 -0.5。
delta_s两个 mesogen 的链序号差。区分相邻链段支持和非局域支持；不再简单排除相邻，而是保留为 local evidence。

四、阈值如何选择

当前默认不是手猜 gb_on=0.30、gb_off=0.12。在推荐的 project pipeline 用法中，只要提供 --gb-param-file 且 --auto-thresholds recommend 保持默认，pipeline 会额外运行 gb_pair_audit.py 生成带抽样清单的候选 pair，再用 lc_calibrate_thresholds.py --calibration-mode joint2d 进行 2D joint calibration。若没有提供 GB 参数文件，则只能回退到已筛选的 edge_diagnostics.tsv，这时不会有 GB audit sampling manifest。

参数当前算法自动应用条件
gb_off_strength在 GB strength × P2 网格中寻找 gray support 足够、背景仍存在的下阈值。必须有足够 candidate pairs、oriented pairs、strong pairs、background pairs 和至少两个独立 frame/block。
gb_on_strength同一 2D 网格中寻找 strong support 足够的上阈值。如果 strong support 不足，只写推荐，不修改主参数。
p2_cut与 GB strength 联合选择，不再只看一维 P2 谷值。如果 event-aware 样本参与校准，自动 apply 会被禁止。
robust_min_s2有 domain diagnostics 时由 domain 内部取向 S2 的分布推荐。样本不足时保留当前值。
重要限制：event-aware sampling 是“事件覆盖增强”，不是严格无偏随机样本。它适合发现拉伸突变、cluster split/merge、near-threshold 情况，但默认不能自动改写 gb_on/off 或 p2_cut；代码会把 apply_allowed=false。

当前代码对所有 sampling 都输出“抽了多少、具体抽了哪些、抽样率”。这是后续判断可靠性的基础。

抽样环节输出文件关键字段

pair sampling for calibrationpair_sampling_manifest.tsvatom_i、atom_j、sampling_strata、candidate_pairs_in_stratum、selected_pairs_in_stratum、pair_sampling_rate。
pair sampling summarypair_sampling_audit.tsv每个 stratum 的估计候选数、被抽中数、抽样率、权重范围、选择原因。

六、domain 到 pearl

domain 和 pearl 不是重复概念。domain 是 pair-contact 图上的局部 mesogen 束；pearl 是多个 robust domain 在 3D 空间上是否构成同一个珠状团块。

判据作用当前状态
pearl_gap_cut两个 domain 的 3D 包络/粒子间最小间隙是否足够小。可自动推荐，但 recommend-only，不自动 apply。
pearl_min_cross_contacts跨 domain 的接触对数量是否足够，避免单根桥接边误合并。由 accepted candidate 的下分位数推荐。
pearl_min_boundary_particles两侧是否都有边界粒子支持接触。用于避免“一个粒子碰一下就合并”。
pearl_max_aspect_ratio合并后是否仍像 bead，而不是细长 connector。aspect ratio 过大说明更像被拉出的丝，不应合成 pearl。

七、主要输出怎么看

OVITO

加载 labeled dump 看 lc_state、lc_domain、lc_pearl。domain 颜色用于看局部束，pearl 颜色用于看空间珠子。halo/edge/mesh 类文件如果 OVITO Basic 不能 Add to scene，可以单独打开或改用 dump 内 label 先审查。

统计图

看 diagnostics 中的阈值分布、domain size vs S2、GB strength vs P2、pearl candidate diagnostics。点太密时优先看 bin summary、joint2d_grid 和抽样 manifest，而不是肉眼看散点。

可靠性

先看 potential 是否 validated，再看 calibration 的 apply_allowed、warnings、sample_sizes、bootstrap CI、joint_2d status。event-aware 参与时不要把推荐值当自动参数。

力学关联

最终用 N_domain(t)、N_pearl(t)、largest fraction、ambiguous fraction、L_parallel、Rg_parallel/perp 等面板解释拉伸竞争。

八、当前仍不应夸大的地方

GB 精确还原：Python GB 重算已经按源码审查修正，但“精确还原 LAMMPS potential”只能在 lc_lammps_run0_validate.py 的 whole-frame run 0 和 sampled two-particle microstates 通过后，对同一 potential/fingerprint 成立。换 potential、pair style、shape 参数、special_bonds/topology 处理方式后需要重验。

Pearl 参数：当前输出自动推荐值和候选表，但不自动应用。原因是 pearl 是更高层结构概念，最好先用 OVITO 和候选表确认误 split/merge 后再固定参数。

Topology：拓扑不是为了研究 bond，而是为了知道哪些 pair 是相邻/局部链段、哪些 pair 可能被 LAMMPS special_bonds 排除，从而避免把链连接本身误解释成非局域聚集。
