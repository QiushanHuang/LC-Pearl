# LC Domain-Pearl V2 Threshold Prior

This folder stores the reusable threshold prior for LC Domain-Pearl V2.

Expected main file:

- `global_thresholds.json`: generated by `scripts/lc_threshold_prior.py`. If it exists, its fingerprint matches the current GB parameter file / mesogen type / topology pair files, and the GB potential is validated, the wrapper applies `gb_off_strength`, `gb_on_strength`, `p2_cut`, `gb_core_strength`, `p2_core_cut`, `gb_strict_core_strength`, and `p2_strict_core_cut` before the main domain/pearl analysis.

Useful audit files generated with it:

- `streaming_manifest.json`: how many files/frames were scanned and how many workers were used.
- `selected_frame_manifest.tsv`: the exact frame files/timesteps used by the streaming prior and their frame sampling weights.
- `stream_hist_2d_gb_p2.tsv`: streaming 2D histogram behind the recommended thresholds.
- `gb_strength_vs_p2_stream_hist.png`: human-readable heatmap with threshold lines and numeric labels.
- `gb_strength_vs_p2_stream_dotgrid.png`: bin-center square dot-grid view of the same 2D histogram.
- `gb_strength_vs_p2_stream_hexbin.png`: true hexbin-style view, closer to the old `gb_strength_vs_p2_hexbin_recommended.png` plot.
- `audit_pair_examples.tsv`: a small reservoir sample of pair rows for manual checking.

Sampling control:

- In `lc_pearl_config.toml`, `threshold_prior.global_frame_stride = 1` means full global-frame sampling. Use `10` or `100` to sample every 10th or 100th global candidate frame. This is the correct control when each dump file contains one frame.
- `threshold_prior.every` only skips frames inside each individual dump file.

You can copy this whole `thresholds/` folder to similar dump folders that use the same Gay-Berne potential and topology exclusion rules. LC-Pearl will rebuild or reject the prior if the fingerprint does not match.
